阶段一

第一次没过破防了决定直接一部到胃一起写必做一和选做一

1. 概述

  1. 根据指导书可知，我们需要实现一级高速缓存和二级告诉缓存，其中l2会多一个脏标签的处理
  2. 公式：C=B* E *S (E缓存块 B字节 S组 C总大小)
    1. l1: E=8 B=64 C=64*1024 S=128
    2. l2: E=8 B=64 C=4*1024 *1024 S=4096
  3. Address:
    1. tag: t bits
    2. set index: s bits
    3. block offset: b bits
    1. l1: b=6 s=7
    2. l2: b=6 s=12

2. 实现

  1. 定义高速缓存的结构(nemu/include/memory/cache.h)
  1. 实现相关函数(nemu/src/memory/cache.c)
  1. 更改hwaddr_read()和hwaddr_write() (emu/src/memory/memory.c)
  1. 由于被多次调用的读写操作函数位于dram.c文件中的，其中有两个函数是用static封装，则需在dram.c文件中进行全局变量的声明(作为接口调用这两个函数)

3. 调试

通过即可

阶段二

模拟IA-32的分段机制

  1. 修改kernel/include/common.h：去掉**#define IA32_SEG*的注释
  2. make run 编译调试，si 1 运行，发现因为无效操作码报错

    如果编译报错如下,则是32位编译环境问题，需要安装必要库
    /usr/include/stdint.h:26:10: fatal error: bits/libc-header-start.h: 没有那个文件或目录
    26 | #include <bits/libc-header-start.h>
    | ^~~~~~~~~~~~~~~~~~~~~~~~~~
    compilation terminated.
    make: *** [lib-common/FLOAT/Makefile.part:16：obj/lib-common/FLOAT/FLOAT.o] 错误 1
    解决方案：
    sudo apt update
    sudo apt install gcc-multilib g++-multilib

  3. 根据报错信息和指导手册，现在需要补全lgdt指令(修改nemu/include/cpu/reg.h)
  4. 段寄存器索引枚举
  1. 定义段寄存器结
  1. GDT描述符读取结构
  1. 定义分段机制相关寄存器
  1. 在restart()函数中初始化分段机制相关的寄存器(修改nemu/src/monitor/monitor.c)
  1. 创建 nemu/include/cpu/exec/data-mov/lgdt.h
  1. 创建 nemu/src/cpu/exec/data-mov/lgdt.c
  1. 创建 nemu/src/cpu/exec/data-mov/lgdt-template.h
  1. 在nemu/src/cpu/exec/exec.c中修改group7的定义
  1. 在nemu/include/cpu/exec/all-instr.h中添加声明
  1. 重新编译

    make clean
    make run
    如果出现以下错误，则为内核编译出现问题

    ar r obj/lib-common/FLOAT/FLOAT.a obj/lib-common/FLOAT/FLOAT.o obj/lib-common/FLOAT/FLOAT_vfprintf.o
